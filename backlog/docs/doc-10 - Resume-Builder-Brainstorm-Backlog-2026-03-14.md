# Resume Builder Brainstorm Backlog

Status: idea capture only
Intent: record brainstormed product directions without turning them into tasks yet
Note: these items are not validated for scope, legality, ethics, or roadmap priority

## AI Workflow And Quality
- Add an AI-enabled pipeline tracker feature that checks whether a job posting is still active.
- Accept user-provided writing samples as additional AI context.
- Add internal self-review that looks for AI tells such as em dashes, repetitive sentence structure, and generic phrasing.
- Offer multiple bullet reframing formats such as STAR and outcome-problem-action.
- Let users save reusable writing rules for resumes and cover letters.
- Add a retry flow with free-text guidance so the next generation round can incorporate specific user feedback.
- Add a bio generator for about-me style fields.

## AI Governance, Limits, And Telemetry
- Add opt-in feedback and telemetry collection.
- Add feature-level usage limits for AI-enabled workflows to prevent extreme usage.
- Make telemetry and AI usage controls legible to users.

## Job Search And Research
- Expose job-search parameters on a dedicated settings page where users can tune skill depth and preferences.
- Add a dedicated research phase for each company covering interview process and likely questions.
- Keep the existing research system flexible enough to support both automated research and user-known interview formats.

## Interview Prep System
- Generate interview prep for each vector.
- Generate separate company-specific prep.
- Generate interview cheat sheets before each interview using company research or user-supplied format details.
- Add a post-interview debrief where users can log questions, their responses, and self-assessment.
- Generate follow-up guidance and prep sheets based on identified gaps.
- Add offer-negotiation prep and cheat sheets.
- Add a what-am-I-worth utility.

## User Data, Privacy, And Portability
- Add export-all-data support.
- Add purge-all-data support.
- Ensure future hosted features preserve strong user control over data lifecycle.

## Portfolio And Microsites
- Generate static microsites as supplements to resumes.
- Allow microsites to include technical deep dives and explanations of how the user works.
- Feed microsite content back into the user profile and authoring context.
- Explore hosting these microsites directly on a Facet-owned domain such as `username.facet.cv`.

## Market Expansion Ideas
- If legal and ethical constraints allow it, explore a database of user-reported interview questions, interview processes, and similar signals.
- Explore whether the product could compete with parts of LinkedIn and Glassdoor by combining job search, company research, interview prep, and shared user intelligence.

## Open Questions For Later Triage
- Which of these ideas are core product extensions versus separate products?
- Which ideas require paid AI entitlements versus free standard functionality?
- Which ideas create meaningful privacy, consent, moderation, or compliance obligations?
- Which ideas belong in the existing resume-builder app versus future hosted account waves?
